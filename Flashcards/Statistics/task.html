Description

Add some statistics features. We suggest you implement the following:

  • The log action saves the application log to the given file. It should save all lines that have been input in or output to the console to the file. You can use a list to store the lines.

  • The hardest card action prints the term of the card that has the most mistakes. You can store the mistake count in a map. If there are no cards with mistakes, you should print There are no cards with errors. For multiple hardest cards, you should list them all like in the example below.

  • The reset stats action erases the mistake counts for all cards.

Also, remember to update serialization and deserialization to store sets of three items (term, definition, mistakes) instead of pairs (term, definition).

Example

The symbol > represents the user input. Notice that it's not the part of the input.

Input the action (add, remove, import, export, ask, exit, log, hardest card, reset stats):

> capitals.txt

Input the action (add, remove, import, export, ask, exit, log, hardest card, reset stats):

Input the action (add, remove, import, export, ask, exit, log, hardest card, reset stats):

> 1
Print the definition of "Russia":
